# NL Plan for Mac — 产品需求文档（PRD）

> **产品名称**：NL Plan（暂定）  
> **平台**：macOS 菜单栏应用  
> **技术栈**：SwiftUI + 原生 macOS  
> **AI 服务**：DeepSeek（架构预留扩展接口）  
> **目标用户**：个人使用  
> **文档版本**：v0.2  
> **日期**：2026-04-16

---

## 一、产品概述

一款 macOS 菜单栏效率工具。用户随时以自然语言输入想法，AI 自动整理为想法池并推荐今日必做项，用户执行任务时自动计时，日终由 AI 给出 S-D 等级评分，最终同步到 macOS 备忘录。

**一句话定义**：把自然语言想法变成可执行的每日任务流，并用 AI 评分驱动自我管理。

---

## 二、核心用户价值

1. **零摩擦记录**：随时打字输入想法，不用手动拆分任务
2. **非阻塞处理**：提交后立即可以继续输入，队列逐个处理
3. **智能排序**：AI 自动整理、分类、推荐
4. **被动计时**：切换任务即自动计时，无需手动操作
5. **游戏化驱动**：每日 S-D 评分，形成正向反馈

---

## 三、核心业务流程

```
用户输入自然语言
       ↓
  输入进入解析队列（非阻塞，可连续输入）
       ↓
  AI 逐个解析（串行处理，并发=1）
       ↓
  队列列表显示处理状态（等待中/解析中/已完成/失败）
       ↓
  点击已完成项 → 进入详情页确认（可编辑/删除/与 AI 对话调整）
       ↓
    想法池 ──────────────────┐
       ↓                      │
  AI 推荐必做项               │ 用户手动挑选
       ↓                      │
    必做项列表 ←──────────────┘
       ↓
  用户点击执行（正计时）
       ↓
  切换任务 → 自动停止上一个 → 启动下一个
       ↓
  日终：AI 评分（S-D）+ 统计
       ↓
  同步到 macOS 备忘录
```

### 关键规则

- **所有任务必须先进入想法池，再进入必做项**（不存在直接创建必做项的路径）
- 想法可随时追加，每次追加经过 AI 整理后进入队列，用户确认后进入想法池
- AI 从想法池中推荐必做项，用户也可以手动从想法池挑选加入必做项
- 执行模式：默认按顺序，支持跳着做、同时进行多个
- **想法池跨天保留**：昨天的想法不会自动清空，每个任务卡片显示创建日期用于区分
- **加入必做项后从想法池消失**：两个池互不重叠，移回时重新出现在想法池
- **已完成任务**：灰色划线显示，自动移到必做项列表底部
- **跨天未完成必做项**：次日启动时自动移回想法池，已开始过的任务标记"已尝试"

## 四、功能模块

### 4.1 自然语言输入

| 项目 | 说明 |
|------|------|
| 输入方式 | 打字输入 |
| 输入时机 | 随时，可多次追加 |
| 输入长度 | 一句话到几句话均可 |
| 非阻塞 | 提交后输入框立即可用，无需等待 AI 返回 |

**交互**：菜单栏点击图标 → 弹出输入框 → 输入后按回车 → 入队处理 → 队列列表展示状态 → 点击已完成项进入详情页确认 → 确认后保存到想法池

### 4.2 解析队列

| 项目 | 说明 |
|------|------|
| 定义 | 用户提交的输入按顺序排队的处理列表 |
| 并发 | 串行处理，同一时间仅 1 个任务在解析 |
| 队列项状态 | waiting（等待中）/ processing（解析中）/ completed（已完成）/ failed（失败） |
| 队列列表位置 | 输入框和想法池之间 |
| 每项显示 | 截断摘要 + 状态图标 + 状态文本（等待中/解析中动画/已完成任务数/失败-点击重试） |
| 详情页 | 点击已完成项进入全屏确认页，含原始输入、解析结果、AI 对话修改、确认/取消按钮 |

### 4.3 AI 解析服务

| 项目 | 说明 |
|------|------|
| 默认 AI | DeepSeek Chat / Reasoner（可在设置页切换） |
| 架构要求 | 预留 AI 服务接口，便于后续扩展其他模型 |
| 调用时机 | 用户提交自然语言时 / 日终评分时 |

**AI 职责**：
1. **解析**：将自然语言整理为结构化任务列表（1-3 个任务，不过度拆分）
2. **分类**：判断每个任务的性质（工作/生活/学习等）
3. **推荐**：推荐其中最应该今天完成的任务
4. **调整**：根据用户对话指令修改已解析的任务列表
5. **评分**：日终根据完成情况给出 S-D 等级评价

**AI 输出格式（想法池阶段）**：
```json
{
  "tasks": [
    {
      "title": "完成需求文档",
      "category": "工作",
      "estimated_minutes": 90,
      "recommended": true,
      "reason": "有明确截止时间，建议优先完成"
    },
    {
      "title": "跑步",
      "category": "生活",
      "estimated_minutes": 45,
      "recommended": false,
      "reason": "非紧急，但建议完成"
    }
  ]
}
```

### 4.4 想法池

| 项目 | 说明 |
|------|------|
| 定义 | 所有经 AI 解析并经用户确认后的任务暂存区 |
| 来源 | 仅通过 AI 解析自然语言生成（经用户确认后进入） |
| 跨天规则 | 想法池永久保留，不自动清空；每个任务显示创建日期用于区分 |
| 信息 | 任务名、分类、预估时长、AI 推荐理由、创建日期 |
| 操作 | 查看、手动加入必做项（加入后从想法池消失）、删除（带内联二次确认） |

### 4.5 必做项

| 项目 | 说明 |
|------|------|
| 来源 | AI 推荐 + 用户从想法池手动挑选 |
| 数量 | 不限，由用户决定 |
| 操作 | 开始执行、调整顺序、移回想法池（重新出现在池中）、标记完成（灰色划线 + 移到底部） |
| 跨天规则 | 未完成的必做项次日自动移回想法池，已开始过的标记"已尝试" |

### 4.6 任务执行与计时

| 项目 | 说明 |
|------|------|
| 计时方式 | 正计时（从 0 开始往上计时） |
| 菜单栏显示 | 正计时时间 + 当前任务名 |
| 切换规则 | 点击新任务 → 自动停止上一个 → 启动新任务 |
| 并行支持 | 可配置允许同时进行多个任务 |
| 打断处理 | 无特殊处理，计时器继续运行 |
| 任务状态 | pending / running / paused / done |

**菜单栏界面示意**：
```
┌─────────────────────────┐
│ ⏱ 00:32:15 写需求文档    │  ← 时钟图标 + 始终可见
├─────────────────────────┤
│ ▶ 写需求文档      00:32  │  ← 当前执行中
│   准备周会材料    预计30m  │
│   跑步           预计45m  │
│   ~~回复邮件~~    已完成   │  ← 灰色划线，列表底部
│   + 添加想法              │
│   📊 今日总结             │
│   📅 历史记录             │
│   ⚙️ 设置                │
└─────────────────────────┘
```

**未配置 API Key 时**：
```
┌─────────────────────────┐
│ 请配置 API               │  ← 替代时钟图标
└─────────────────────────┘
```

### 4.7 日终评分

**触发时机**：
1. 用户手动点击"结束今天"按钮
2. 次日首次打开应用时，自动检查并补评昨日（如昨日未评分）

> 不依赖定时任务（如固定晚上 11:30），因为用户可能已关机或合盖。

**评分维度**：
- 必做项完成率
- 每个必做项的实际时长 vs 预估时长偏差
- 想法池中额外完成的任务（加分项）

**评分申诉机制**：
- 用户对评分不满意时，可点击"驳斥"按钮
- AI 展示本次评分的理由和依据
- 用户可在输入框内输入自己的想法告诉 AI
- AI 根据用户反馈重新评分
- **每天限 3 次申诉机会**

**等级定义**：

| 等级 | 含义 | 大致标准 |
|------|------|----------|
| S | 完美的一天 | 必做项全部完成，时间偏差小，额外完成想法池任务 |
| A | 优秀 | 必做项全部完成，时间基本合理 |
| B | 良好 | 必做项完成 80%+ |
| C | 一般 | 必做项完成 50%+ |
| D | 需要改进 | 必做项完成不到 50% |

**评分输出格式**：
```json
{
  "grade": "A",
  "summary": "今天完成了 4/5 个必做项，效率不错！",
  "grading_basis": "评分依据和理由",
  "stats": {
    "total_tasks": 5,
    "completed_tasks": 4,
    "total_planned_minutes": 240,
    "total_actual_minutes": 265,
    "deviation_rate": 0.1
  },
  "suggestion": "明天建议优先处理未完成的周会材料。"
}
```

### 4.8 同步到备忘录

| 项目 | 说明 |
|------|------|
| 目标 | macOS 自带备忘录（Apple Notes） |
| 内容 | 日终评分 + 统计数据 |
| 方式 | V1 先通过 AppleScript 或导出方式实现 |
| 时机 | 日终评分完成后自动同步 |

## 五、数据模型

### 5.1 Thought（想法/自然语言输入）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| raw_text | String | 用户原始输入文本 |
| created_at | Date | 输入时间 |
| processed | Bool | 是否已被 AI 解析 |

### 5.2 Task（任务）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| title | String | 任务名称 |
| category | String | 分类（工作/生活/学习等） |
| estimated_minutes | Int | AI 预估时长 |
| ai_recommended | Bool | AI 是否推荐为必做 |
| recommendation_reason | String | AI 推荐理由 |
| pool | Enum | 所在池（idea_pool / must_do） |
| sort_order | Int | 排序权重 |
| status | Enum | pending / running / paused / done |
| date | Date | 任务所属日期（跨天迁移后可能变化） |
| created_date | Date | 创建日期（用于想法池中按日区分，不变） |
| attempted | Bool | 是否曾经尝试执行过（跨天移回想法池时标记） |

### 5.3 ParseQueueItem（解析队列项）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| rawText | String | 用户原始输入文本 |
| status | Enum | waiting / processing / completed / failed |
| parsedTasks | [ParsedTask]? | AI 解析结果（completed 时非空） |
| errorMessage | String? | 失败原因（failed 时非空） |
| createdAt | Date | 入队时间 |

> ParseQueueItem 为内存中的 @Observable 对象，不持久化。应用重启后队列清空。

### 5.4 SessionLog（计时记录）

> 一个任务可对应多条 SessionLog，每次执行新增一条记录，跨天执行时各自独立。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| task_id | UUID | 关联任务 |
| started_at | Date | 开始时间 |
| ended_at | Date? | 结束时间（null 表示仍在进行） |
| duration_seconds | Int | 时长（秒） |
| date | Date | 执行日期（用于按天聚合统计） |

### 5.5 DailySummary（日终总结）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| date | Date | 日期 |
| grade | String | S/A/B/C/D |
| summary | String | AI 评价文本 |
| suggestion | String | 明日建议 |
| total_planned_minutes | Int | 计划总时长 |
| total_actual_minutes | Int | 实际总时长 |
| completed_count | Int | 完成任务数 |
| total_count | Int | 总任务数 |
| synced_to_notes | Bool | 是否已同步到备忘录 |
| appeal_count | Int | 已申诉次数（每日限 3 次） |
| grading_basis | String | AI 评分理由和依据（申诉时展示） |

## 六、界面结构

### 6.1 菜单栏常驻

- 时钟图标 + 当前正计时 + 当前任务名
- 未配置 API Key 时显示"请配置 API"
- 点击展开主面板

### 6.2 主面板（Popover）

- **输入区**：文本输入框 + 提交按钮（提交后立即可用）
- **解析队列**：队列项列表，显示处理状态，点击已完成项进入详情页
- **想法池**：折叠展示，可展开查看所有想法，支持手动加入必做项
- **必做项列表**：当前任务列表，支持排序、开始执行
- **底部操作**：今日总结、历史、设置

### 6.3 解析详情页

- 全屏页面（占满 Popover，无底部工具栏）
- 左上角返回按钮
- 用户原始输入
- AI 解析结果列表（支持编辑/删除/与 AI 对话修改）
- 确认添加 / 取消按钮（AI 处理期间锁定）

### 6.4 今日总结页

- AI 评分等级（大字展示 S/A/B/C/D）
- 统计数据可视化
- AI 评价文本
- **驳斥按钮**：点击后展示 AI 评分理由，用户可输入想法要求重新评分（每日限 3 次，用完后按钮变灰不可点）
- 一键同步到备忘录

### 6.5 历史记录页

- **日历视图**：每格显示日期 + 当天评分等级（S/A/B/C/D），未评分的天显示"—"
- 点击某天可查看该天的详细总结

### 6.6 设置页

- **API Key 配置**：DeepSeek API Key 输入框（必填，存储于 UserDefaults + Base64 编码）
- **AI 模型选择**：DeepSeek Chat / DeepSeek Reasoner
- **外观模式**：跟随系统 / 浅色 / 深色
- 并行任务开关
- 备忘录同步开关
- 数据导出

## 七、AI 提示词设计（方向）

### 7.1 想法解析 Prompt

```
你是一个任务管理助手。用户会用自然语言描述今天的想法和计划。
请将用户的输入整理为结构化的任务列表。

要求：
1. 每个任务必须是可执行、可完成的具体行动
2. 将内容相近的细碎事项合并为一个任务，保持每个任务有足够的体量（30-120 分钟）
3. 为每个任务预估合理时长（分钟）
4. 推荐其中最应该今天完成的任务（recommended = true）
5. 为每个任务分类（工作/生活/学习/健康/其他）
6. 生成 1-3 个任务即可，不要过度拆分
```

### 7.2 解析结果修改 Prompt

```
你是一个任务管理助手。用户之前输入了一段想法，你将其解析为了任务列表。
现在用户希望对解析结果进行调整。
请根据用户的修改要求，调整任务列表，输出修改后的完整任务列表。
```

### 7.3 日终评分 Prompt

```
你是一个效率教练。根据用户今天的任务完成情况，给出评价和评分。

评分标准：
- S：必做项全部完成，时间偏差 ≤10%，额外完成想法池任务
- A：必做项全部完成，时间偏差 ≤20%
- B：必做项完成 ≥80%
- C：必做项完成 ≥50%
- D：必做项完成 <50%

要求：
1. 给出等级和具体评价
2. 指出做得好的地方
3. 指出可以改进的地方
4. 给出明日建议

输出严格的 JSON 格式。
```
