# NL Plan for Mac — 产品需求文档

## 1. 产品定位

一款 macOS 菜单栏 AI 计划管理工具。用户用自然语言描述计划，AI 自动拆解为结构化任务，配合计时、项目管理和日终评分，形成「输入→规划→执行→复盘」的完整闭环。

## 2. 核心用户场景

### 2.1 日常规划

用户在菜单栏输入「今天下午要把报告写完，顺便回复三封邮件，晚上跑个步」，AI 自动拆解为独立任务，带上分类、预估时长，进入想法池。用户挑选合适的加入今日必做项，启动计时执行。

### 2.2 长期项目推进

用户将一个大型目标（如「搭建个人博客」）标记为项目。系统通过项目说明、规划背景、项目备注和绑定必做项的完成情况，在每次 AI 推荐时生成项目状态摘要，推荐下一步最合适的切片任务。

### 2.3 日终复盘

一天结束时，用户触发日终结算。AI 根据完成率、时间偏差、任务详情和用户备注给出 A-F 评分和改进建议。用户可以驳斥评分（最多 3 次），AI 会重新评估。

## 3. 功能模块

### 3.1 自然语言输入

| 需求 | 说明 |
|------|------|
| 输入方式 | 文本框，支持多行，最多 4000 字 |
| 解析流程 | 提交后进入解析队列，AI 逐个拆解为结构化任务 |
| 非阻塞 | 提交后可继续输入，队列后台处理 |
| 确认机制 | 解析完成后用户可逐条编辑标题/分类/预估时长，确认后入库 |
| 二次修改 | 对解析结果不满意可追加指令让 AI 重新调整 |

### 3.2 想法池

| 需求 | 说明 |
|------|------|
| 任务列表 | 展示所有待处理/已尝试的想法和项目 |
| 项目标记 | 想法可标记为「项目」，项目无整体预估时长，通过绑定必做项切片推进 |
| 项目详情 | 展示项目说明、规划背景、备注列表、活跃推进任务和已归档推进记录 |
| 编辑能力 | 修改标题、分类、备注、项目说明、规划背景 |
| 来源绑定 | 必做项可绑定到项目想法，实现项目任务追踪 |
| AI 清理 | AI 分析想法池，推荐删除不合适的条目，支持批量操作和撤销 |
| 搜索过滤 | 按标题搜索，按分类标签过滤 |

### 3.3 必做项（今日任务）

| 需求 | 说明 |
|------|------|
| 任务列表 | 展示今日必做项，按优先级分组排序 |
| 优先级 | 高/中/低三级，同优先级内可拖拽排序 |
| 计时器 | 同一时刻只有一个任务可计时（也可开启并行计时） |
| 完成标记 | 双击确认完成任务 |
| 退回想法池 | 不想做的必做项可退回想法池 |
| 来源追踪 | 每条必做项记录来源想法（普通想法/项目/无来源） |

### 3.4 AI 推荐

| 需求 | 说明 |
|------|------|
| 推荐模式 | 快速（清理想法池）和综合（平衡普通想法和项目） |
| 快速模式 | 优先推荐普通想法，不走项目摘要准备，减少 token 和等待 |
| 综合模式 | 普通想法和项目都参与，项目会先刷新推荐状态摘要 |
| 推荐数量 | 1-3 条，附推荐理由和优先级建议 |
| 接受操作 | 逐条接受或一键全部接受 |
| 项目切片 | 项目推荐时 AI 生成具体可执行的切片任务，而非推荐整个项目 |

### 3.5 日终评分

| 需求 | 说明 |
|------|------|
| 触发时机 | 用户手动结束今天 |
| 评分范围 | A-F 六级，附评分依据 |
| 未完成任务 | 必须填写备注才能结算 |
| 评分内容 | 总结、改进建议、统计数据（完成率、时间偏差） |
| 驳斥机制 | 最多 3 次，用户提交反馈后 AI 重新评分 |
| 历史查看 | 按月历查看每日评分和详情 |

### 3.6 跨天处理

| 需求 | 说明 |
|------|------|
| 自动迁移 | 昨日未完成的必做项自动移回想法池 |
| 补结算 | 昨日未评分时，今天会提示补结算 |
| 归档 | 结算后的必做项标记为已归档（isSettled），不删除 |

## 4. 数据模型

### 4.1 核心实体

| 实体 | 用途 |
|------|------|
| IdeaEntity | 想法池条目（普通想法 + 项目） |
| DailyTaskEntity | 今日必做项（活跃 + 已归档） |
| DailySummaryEntity | 日终评分记录 |
| SessionLogEntity | 计时记录 |
| ThoughtEntity | 用户原始输入文本 |
| ProjectNoteEntity | 项目备注 |

### 4.2 关键关系

- IdeaEntity ↔ DailyTaskEntity：通过 `sourceIdeaId` 关联，一个想法可关联多条必做项
- DailyTaskEntity ↔ SessionLogEntity：通过 `taskId` 关联，记录每次计时的开始/结束
- IdeaEntity ↔ ProjectNoteEntity：通过 `ideaId` 关联，项目可有多条备注
- DailyTaskEntity 的 `isSettled` 区分活跃任务和归档记录

## 5. AI 能力清单

| 能力 | 触发时机 | 输入 | 输出 |
|------|----------|------|------|
| 自然语言解析 | 用户提交输入 | 原始文本 + 已有标题 | 结构化任务列表 |
| 任务修改 | 用户追加指令 | 原始输入 + 当前结果 + 指令 | 调整后的任务列表 |
| 项目分类 | 用户确认解析结果 | 任务标题 + 分类 | 是否为项目 |
| 推荐必做项 | 用户请求推荐 | 想法池 + 必做项 + 剩余时间 | 1-3 条推荐 |
| 项目状态摘要 | 综合推荐前自动触发 | 项目上下文 | 压缩摘要 |
| 项目进度分析 | 用户手动刷新 | 绑定任务完成情况 | 进度百分比 + 摘要 |
| 规划背景提示词 | 用户在项目详情点击生成 | 项目上下文 | 研究提示词 |
| 想法池清理 | 用户请求清理 | 想法池列表 | 建议删除条目 |
| 日终评分 | 用户结束今天 | 任务完成详情 + 备注 | A-F 评分 + 总结 |
| 评分驳斥 | 用户不满评分 | 原始评分 + 用户反馈 | 新评分 |
