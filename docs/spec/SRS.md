# NL Plan for Mac — 软件需求规格说明

## 1. 引言

本文档描述 NL Plan for Mac 的功能需求和非功能需求。所有需求基于当前实际运行的代码。

## 2. 功能需求

### 2.1 自然语言任务输入

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-INPUT-01 | 用户在文本框中输入自然语言描述，系统通过 AI 解析为结构化任务（标题、分类、预估时长） | 必须 |
| F-INPUT-02 | 提交后不阻塞界面，任务进入解析队列后台处理 | 必须 |
| F-INPUT-03 | 解析完成后用户可逐条编辑任务属性（标题、分类、预估时长） | 必须 |
| F-INPUT-04 | 用户可对解析结果追加自然语言指令，AI 重新调整结果 | 必须 |
| F-INPUT-05 | AI 自动判断任务是否属于项目型想法 | 必须 |
| F-INPUT-06 | 输入文本长度上限 4000 字 | 必须 |

### 2.2 想法池管理

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-POOL-01 | 想法池展示所有待处理、已尝试的想法和项目 | 必须 |
| F-POOL-02 | 用户可将想法标记为项目，项目无整体预估时长 | 必须 |
| F-POOL-03 | 项目支持编辑项目说明和规划背景，供 AI 推荐时参考 | 必须 |
| F-POOL-04 | 项目支持添加、编辑备注记录 | 必须 |
| F-POOL-05 | 项目详情展示活跃推进任务和已归档推进记录 | 必须 |
| F-POOL-06 | 用户可手动将想法提升为必做项，选择优先级 | 必须 |
| F-POOL-07 | 用户可删除想法池中的条目 | 必须 |
| F-POOL-08 | 支持按标题搜索和按分类标签过滤 | 必须 |
| F-POOL-09 | AI 可分析想法池，推荐删除不合适的条目（支持批量操作和撤销） | 必须 |
| F-POOL-10 | AI 可为项目生成规划研究提示词，辅助用户完善规划背景 | 必须 |

### 2.3 必做项管理

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-MUST-01 | 展示今日必做项，按优先级（高→中→低）分组排序 | 必须 |
| F-MUST-02 | 同优先级内支持上移/下移排序 | 必须 |
| F-MUST-03 | 用户可启动计时执行任务，菜单栏实时显示进度 | 必须 |
| F-MUST-04 | 默认单任务计时，可在设置中开启并行计时 | 必须 |
| F-MUST-05 | 用户可标记任务完成（双击确认） | 必须 |
| F-MUST-06 | 用户可将必做项退回想法池 | 必须 |
| F-MUST-07 | 必做项记录来源想法 ID 和来源类型（普通想法/项目/无来源） | 必须 |
| F-MUST-08 | 用户可为必做项重新绑定来源项目 | 必须 |
| F-MUST-09 | 每条必做项展示累计计时时间 | 必须 |

### 2.4 AI 推荐必做项

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-REC-01 | 支持两种推荐模式：快速和综合 | 必须 |
| F-REC-02 | 快速模式优先推荐普通想法，不走项目摘要准备链路 | 必须 |
| F-REC-03 | 综合模式同时考虑普通想法和项目 | 必须 |
| F-REC-04 | 综合模式下，项目在推荐前自动刷新推荐状态摘要（如过期） | 必须 |
| F-REC-05 | 推荐结果 1-3 条，包含标题、分类、预估时长和推荐理由 | 必须 |
| F-REC-06 | 项目推荐时生成具体可执行的切片任务 | 必须 |
| F-REC-07 | 用户可调整推荐项优先级 | 必须 |
| F-REC-08 | 用户可逐条或一键全部接受推荐 | 必须 |
| F-REC-09 | 多个项目的摘要生成支持并行（上限 2 个并发） | 必须 |
| F-REC-10 | 单个项目摘要生成失败不阻塞整轮推荐 | 必须 |

### 2.5 日终评分

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-GRADE-01 | 用户手动触发日终结算 | 必须 |
| F-GRADE-02 | AI 根据完成率、时间偏差、任务详情给出 A-F 评分 | 必须 |
| F-GRADE-03 | 评分包含总结、改进建议和评分依据 | 必须 |
| F-GRADE-04 | 未完成任务需填写备注后才能结算 | 必须 |
| F-GRADE-05 | 用户可驳斥评分（最多 3 次），AI 根据反馈重新评分 | 必须 |
| F-GRADE-06 | AI 不可用时降级为基础规则评分 | 必须 |

### 2.6 结算与归档

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-SETTLE-01 | 结算时必做项标记为已归档（isSettled=true），计算实际耗时 | 必须 |
| F-SETTLE-02 | 已归档必做项不在活跃列表中显示，但在项目详情中可见 | 必须 |
| F-SETTLE-03 | 完成的必做项对应的想法状态更新为已完成 | 必须 |
| F-SETTLE-04 | 未完成的必做项对应的想法状态更新为已尝试 | 必须 |
| F-SETTLE-05 | 项目无活跃绑定任务时状态回退为待处理 | 必须 |
| F-SETTLE-06 | 结算后刷新项目推荐上下文时间 | 必须 |

### 2.7 跨天处理

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-DAY-01 | 昨日未完成的必做项自动移回想法池 | 必须 |
| F-DAY-02 | 昨日未评分时，今天提示补结算 | 必须 |
| F-DAY-03 | 支持结算历史日期 | 必须 |

### 2.8 历史记录

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-HIST-01 | 按月历展示每日评分，颜色标识等级 | 必须 |
| F-HIST-02 | 点击日期查看当日评分详情和统计数据 | 必须 |

### 2.9 设置

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-SET-01 | 配置 DeepSeek API Key | 必须 |
| F-SET-02 | 选择 AI 模型 | 必须 |
| F-SET-03 | 设置工作结束时间（影响剩余时间计算） | 必须 |
| F-SET-04 | 开关并行计时模式 | 必须 |
| F-SET-05 | 开机自启动 | 必须 |
| F-SET-06 | 管理自定义分类标签 | 必须 |

## 3. 非功能需求

| 编号 | 需求 | 说明 |
|------|------|------|
| NF-01 | 本地运行 | 所有数据存储在本地，不依赖云服务（AI API 除外） |
| NF-02 | 菜单栏集成 | 以 macOS 菜单栏图标形式常驻，不占用 Dock 位置 |
| NF-03 | 系统版本 | macOS 14.0 (Sonoma) 及以上 |
| NF-04 | 数据迁移 | SwiftData 自动轻量迁移，新增字段需有默认值 |
| NF-05 | AI 容错 | 所有 AI 调用具备重试机制，失败时有降级方案 |
| NF-06 | 并发安全 | TimerEngine 和 AIExecutionCoordinator 使用 actor 隔离 |
| NF-07 | 响应速度 | 快速推荐模式减少 AI 请求次数和 token 消耗 |

## 4. 数据实体关系

```
ThoughtEntity ──── 1:1 ──── IdeaEntity (通过 saveParsedTasks 关联)
IdeaEntity ──── 1:N ──── DailyTaskEntity (通过 sourceIdeaId)
IdeaEntity ──── 1:N ──── ProjectNoteEntity (通过 ideaId)
DailyTaskEntity ──── 1:N ──── SessionLogEntity (通过 taskId)
DailySummaryEntity (独立，按日期索引)
```

## 5. 外部依赖

| 依赖 | 用途 | 说明 |
|------|------|------|
| DeepSeek API | AI 能力 | 自然语言解析、推荐、评分等 |
| Keychain | API Key 存储 | 安全存储用户凭证 |
| SwiftData | 数据持久化 | Apple 原生 ORM |
